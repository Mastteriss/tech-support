1. ОБЪЕКТ ИСПЫТАНИЙ

Объектом испытаний является программное обеспечение 
"Система технической поддержки", представляющее собой 
клиент-серверное приложение для автоматизации работы 
службы технической поддержки.

Программа состоит из следующих частей:
- серверная часть (Flask-приложение, API);
- клиентская часть (графический интерфейс на Tkinter);
- база данных (SQLite);
- модульные тесты (unittest).


2. ЦЕЛЬ ИСПЫТАНИЙ

Целью проведения испытаний является проверка соответствия 
программного обеспечения требованиям, установленным в 
техническом задании, а также выявление и устранение ошибок 
в работе программы.

Основные задачи испытаний:
1. Проверка корректности работы всех функций системы.
2. Проверка устойчивости к некорректному вводу данных.
3. Проверка сохранности данных при перезапуске.
4. Проверка взаимодействия клиента и сервера.
5. Проверка выполнения требований к производительности.


3. ТРЕБОВАНИЯ К ПРОГРАММЕ

3.1. Функциональные требования:
- регистрация вопросов клиентов;
- назначение консультантов на вопросы;
- добавление ответов на вопросы;
- просмотр списка вопросов с фильтрацией по статусу;
- просмотр статистики работы;
- добавление новых консультантов.

3.2. Требования к интерфейсу:
- интуитивно понятный графический интерфейс;
- отображение статуса каждого вопроса;
- разделение на функциональные вкладки.

3.3. Требования к надежности:
- устойчивость к некорректному вводу данных;
- сохранение данных в базе SQLite;
- корректная обработка ошибок подключения к серверу.

3.4. Требования к производительности:
- время отклика API не более 1 секунды;
- загрузка списка вопросов не более 2 секунд;
- поддержка работы с 1000+ вопросами.

5. СОСТАВ И ПОРЯДОК ПРОВЕДЕНИЯ ИСПЫТАНИЙ

Испытания проводятся в три этапа:

5.1. Этап 1. Модульное тестирование (автоматизированное)
Проверка отдельных компонентов программы:
- тестирование моделей данных (test_models.py);
- тестирование API эндпоинтов (test_api.py);
- тестирование работы с базой данных.

5.2. Этап 2. Интеграционное тестирование (полуавтоматическое)
Проверка взаимодействия компонентов:
- клиент ↔ сервер;
- сервер ↔ база данных;
- пользовательский интерфейс ↔ API.

5.3. Этап 3. Приемочные испытания (ручное тестирование)
Проверка всех функций программы пользователем:
- создание вопроса;
- назначение консультанта;
- добавление ответа;
- отметка вопроса как решенного;
- просмотр статистики.


6. МЕТОДЫ ИСПЫТАНИЙ

6.1. Метод модульного тестирования
- Тестирование методом "белого ящика" (White Box Testing)
- Использование фреймворка unittest
- Проверка каждого метода на корректность

Пример теста для модели Question:
def test_assign_consultant(self):
question = Question(1, "Тест", "Описание", "Клиент")
question.assign_consultant(5)
assert question.consultant_id == 5
assert question.status == Question.STATUS_IN_PROGRESS

6.2. Метод API тестирования
- Тестирование методом "черного ящика" (Black Box Testing)
- Использование test_client из Flask
- Проверка HTTP статусов и формата ответов

6.3. Метод ручного функционального тестирования
- Выполнение всех пользовательских сценариев
- Проверка GUI на корректность отображения
- Тестирование граничных условий

Сценарии ручного тестирования:
1. Создание вопроса с некорректными данными (пустые поля)
2. Создание вопроса с корректными данными
3. Назначение несуществующего консультанта
4. Добавление ответа без назначенного консультанта
5. Решение вопроса с открытыми ответами
6. Фильтрация вопросов по статусу


7. СРЕДСТВА И ПОРЯДОК ПРОВЕДЕНИЯ ИСПЫТАНИЙ

7.1. Аппаратные средства:
- Процессор: Intel Core i3 или выше;
- ОЗУ: не менее 2 ГБ;
- Свободное место на диске: 100 МБ.

7.2. Программные средства:
- Операционная система: Windows/Linux/macOS;
- Python 3.8 или выше;
- PyCharm/VS Code (для разработки);
- SQLite Browser (для просмотра БД).

7.3. Порядок проведения:

7.3.1. Запуск модульных тестов:
cd server
python -m unittest test_models.py
python -m unittest test_api.py
7.3.2. Запуск интеграционных тестов:
Запустить сервер
python server/app.py

Запустить клиент
python client/client.py

Выполнить сценарии взаимодействия
7.3.3. Проведение ручного тестирования:
Последовательность действий:
Запустить сервер
Запустить клиент
Создать 5-10 вопросов
Добавить 2-3 консультантов
Назначить консультантов на вопросы
Добавить ответы
Отметить часть вопросов как решенные
Проверить фильтрацию
Проверить статистику
Перезапустить систему и убедиться в сохранности данных


8. УСЛОВИЯ ПРОВЕДЕНИЯ ИСПЫТАНИЙ

8.1. Требования к персоналу:
- Проводящий испытания должен иметь базовые знания Python
- Знать структуру проекта и назначение компонентов
- Уметь работать с командной строкой

8.2. Требования к окружению:
- Наличие интерпретатора Python
- Установленные зависимости (Flask, flask-cors, requests)
- Доступ к портам 5000 (для сервера)

8.3. Требования к тестовым данным:
- Использовать тестовые данные, не влияющие на реальную работу
- После тестирования очищать базу данных


9. ОБРАБОТКА РЕЗУЛЬТАТОВ

9.1. Критерии успешности модульных тестов:
- Все тесты должны выполняться без ошибок (OK)
- Процент успешных тестов = 100%
- Время выполнения тестов не более 5 секунд

9.2. Критерии успешности API тестов:
- Все эндпоинты отвечают корректными HTTP кодами
- Данные сохраняются и извлекаются правильно
- Обработка ошибок работает корректно

9.3. Критерии успешности ручных тестов:
- Все функции выполняются без ошибок
- Интерфейс интуитивно понятен
- Данные сохраняются между сессиями
- Программа корректно обрабатывает ошибки ввода
