1. ОБЩИЕ СВЕДЕНИЯ

Настоящая спецификация устанавливает состав программного 
обеспечения "Система технической поддержки", его компоненты, 
взаимосвязи и требования к среде выполнения.

Полное наименование: Программное обеспечение "Система технической поддержки"
Краткое наименование: TechSupport
Версия: 1.0
Дата создания: 2024 г.


2. СОСТАВ ПРОГРАММНОГО КОМПЛЕКСА

Программный комплекс состоит из следующих частей:

2.1. Серверная часть (Server Components)
2.2. Клиентская часть (Client Components)
2.3. База данных (Database)
2.4. Тесты (Test Components)
2.5. Документация (Documentation)


3. ОПИСАНИЕ КОМПОНЕНТОВ

3.1. Серверная часть

3.1.1. Файл: server/app.py
Назначение: Главный модуль сервера, реализующий REST API
Функции:
- Запуск Flask-приложения
- Обработка HTTP-запросов
- Взаимодействие с сервисным слоем
Зависимости: Flask, flask-cors, models, database

Интерфейс API:
GET    /api/health               - проверка работоспособности
GET    /api/consultants          - список консультантов
POST   /api/consultants          - создание консультанта
GET    /api/questions            - список вопросов
GET    /api/questions/<id>       - вопрос по ID
POST   /api/questions            - создание вопроса
PUT    /api/questions/<id>/assign - назначение консультанта
PUT    /api/questions/<id>/resolve - отметка как решенного
POST   /api/answers              - добавление ответа
GET    /api/statistics           - статистика

3.1.2. Файл: server/models.py
Назначение: Модели данных предметной области
Классы:
- Consultant      - модель консультанта
- Question        - модель вопроса
- Answer          - модель ответа
- TechSupportService - сервис управления данными

Методы:
Consultant:
  - __init__()           - инициализация
  - to_dict()            - преобразование в словарь
  - from_dict()          - создание из словаря
  - increment_handled()  - увеличение счетчика обработанных вопросов

Question:
  - __init__()           - инициализация
  - assign_consultant()  - назначение консультанта
  - add_answer()         - добавление ответа
  - resolve()            - отметка как решенного
  - close()              - закрытие вопроса
  - to_dict()            - преобразование в словарь
  - from_dict()          - создание из словаря

TechSupportService:
  - load_data()          - загрузка данных из БД
  - get_consultants()    - получение списка консультантов
  - add_consultant()     - добавление консультанта
  - get_questions()      - получение списка вопросов
  - create_question()    - создание вопроса
  - assign_consultant_to_question() - назначение консультанта
  - add_answer()         - добавление ответа
  - resolve_question()   - отметка вопроса как решенного
  - get_statistics()     - получение статистики

3.1.3. Файл: server/database.py
Назначение: Работа с базой данных SQLite
Классы:
- Database           - управление БД

Методы:
- __init__()          - инициализация и создание таблиц
- get_consultants()   - чтение всех консультантов
- save_consultant()   - сохранение консультанта
- update_consultant() - обновление консультанта
- get_questions()     - чтение всех вопросов
- save_question()     - сохранение вопроса
- update_question()   - обновление вопроса
- get_answers()       - чтение всех ответов
- save_answer()       - сохранение ответа
- clear_all()         - очистка БД (для тестов)

Структура таблиц:
consultants:
  - id INTEGER PRIMARY KEY
  - name TEXT NOT NULL
  - specialization TEXT NOT NULL
  - email TEXT NOT NULL UNIQUE
  - questions_handled INTEGER DEFAULT 0

questions:
  - id INTEGER PRIMARY KEY
  - title TEXT NOT NULL
  - description TEXT NOT NULL
  - client_name TEXT NOT NULL
  - priority INTEGER DEFAULT 1
  - status TEXT DEFAULT 'new'
  - created_at TEXT NOT NULL
  - updated_at TEXT NOT NULL
  - consultant_id INTEGER FOREIGN KEY

answers:
  - id INTEGER PRIMARY KEY
  - question_id INTEGER NOT NULL
  - consultant_id INTEGER NOT NULL
  - content TEXT NOT NULL
  - created_at TEXT NOT NULL


3.2. Клиентская часть

3.2.1. Файл: client/client.py
Назначение: Графический интерфейс пользователя
Классы:

4. ЗАВИСИМОСТИ

4.1. Внешние зависимости (требуют установки):
- Python 3.8+
- Flask 2.3.2 - веб-фреймворк
- flask-cors 4.0.0 - поддержка CORS
- requests 2.31.0 - HTTP клиент

4.2. Внутренние зависимости:
- server/models.py ← server/database.py
- server/app.py ← server/models.py, server/database.py
- client/client.py → server/app.py (через API)

4.3. Системные зависимости:
- Tkinter (входит в состав Python)
- SQLite3 (входит в состав Python)


5. ТРЕБОВАНИЯ К СРЕДЕ ВЫПОЛНЕНИЯ

5.1. Минимальные требования:
- Операционная система: Windows 7/10, macOS 10.14+, Linux
- Процессор: 1 ГГц
- ОЗУ: 1 ГБ
- Свободное место на диске: 100 МБ
- Интерпретатор: Python 3.8+

5.2. Рекомендуемые требования:
- Операционная система: Windows 10/11, macOS 12+, Ubuntu 20.04+
- Процессор: 2 ГГц
- ОЗУ: 4 ГБ
- Свободное место: 500 МБ
- IDE: PyCharm или VS Code

5.3. Сетевое окружение:
- Локальный запуск: порт 5000 для сервера
- Сетевой запуск: доступность порта 5000
